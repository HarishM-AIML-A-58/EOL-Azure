# Design System Governance & Contribution

The TendWorks Design System is a collaborative effort. This document outlines how to contribute and how we manage changes.

## 1. Contribution Workflow

1.  **Issue/Request**: Open a ticket for a new component or bug fix.
2.  **Exploration**: Designers create mockups in the Design System Figma file.
3.  **Review**: Weekly design/dev sync to approve the new pattern.
4.  **Implementation**: Develop the component in the library following established API standards.
5.  **Documentation**: Update `components.md` and live docs.

## 2. Versioning Strategy

We follow **Semantic Versioning (SemVer)**:
*   **MAJOR**: Breaking changes (e.g., removing a prop, changing token names).
*   **MINOR**: New components or features (backward compatible).
*   **PATCH**: Bug fixes or visual tweaks.

## 3. Communication Channels

*   **#design-system**: Slack channel for general questions.
*   **Monthly Newsletter**: Summary of updates, new components, and deprecations.

## 4. Support

If you encounter an issue or have a question:
1.  Check the documentation first.
2.  Search the existing issues.
3.  Reach out in the Slack channel.
